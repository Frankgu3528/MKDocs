    堆

    11-03 智云

    二叉堆

    • 完全二叉树
    • 父节点>子节点（大根堆）这样堆顶就是最大值 根据完全二叉树的性质，第\(i\)个节点的左孩子就是\(2i\)，右孩子就是\(2i+1\)。所以可以直接用一个数组来存。(\(a_1\)就是堆顶元素，依次排下去)

    插入

    插入一个新元素，先插到最后，然后再不停换（升堆） 向上调整：如果这个结点的权值大于它父亲的权值，就交换，重复此过程直到不满足或者到根。复杂度\(O(logn)\)

    void up(int x) {
  while (x > 1 && h[x] > h[x / 2]) {
    swap(h[x], h[x / 2]);
    x /= 2;
  }
}

    PS: 既然我们可以用一个数组形式地去记录一个堆,那么其实每次交换可以不实际地去做三次赋值，而是用\(a_0\)这个元素去“缓存”下你要交换的值，这样可以减少一些赋值操作，提高效率。

    删除操作

    删除操作指删除堆中最大的元素，即删除根结点。但是如果直接删除，则变成了两个堆，难以处理。 我们的做法是：把堆顶的节点与最后一个节点进行交换，然后删掉他，最后对堆顶元素向下调整，就能使调整完的东西重新满足堆的性质。

    建堆

    上面讲了堆的一些基本操作，那给你一个数组，怎么建立一个堆呢? 1. 一个一个insert是最简单的方法，但这样的复杂度是\(O(nlogn)\)。 下面是常用的\(O(n)\)建堆的方法： 从约为 \(n/2\) 的地方开始调整（因为后面的节点都是叶子节点，无需调整），从后往前对每个节点进行向下调整，这个过程中，需要比较和交换的节点个数，跟这个节点的高度 k 成正比。 所以只要求倒数第二层节点往前的高度，注意这是一个等差 * 等比的式子，计算后就是 \(2^{h+1}-h-2\)，代入h = log2n, 就得到复杂度是 \(O(n)\) 。 比较清楚的推导

    堆排序

    先建立一个大根堆，再进行一次DELETE操作，可以得到整个堆的最大值，将剩余n-1个元素重新调整成堆后，在一次DELETE得到次大值，如此就能得到最后有序的结果。

    时间复杂度

    \(O(nlogn)\) （每次DELETE都是\(O(logn)\)） 堆本来是非原地算法。not-in-place ，但用数组模拟堆可以做到原地。（堆是个幻觉）

    堆排序进阶

    要求你输出第k大的元素？ 方案1：进行k次DELETE操作，就是要的结果。 （这样最坏效率是\(O(nlogn)\)） 方案2：给你n个元素\(a_1,a_2,…,a_n\)，先给前k个建最小堆。然后

    for i = k+1 to n:
    if (a[i] > 堆顶元素) :
        insert(a[i])

    这样堆里的k-1个元素都比最后的堆顶大，而没进堆的都比堆顶小，这样最后的堆顶就是第 \(k\) 大的。 上面只是提供一种新颖的思路，但似乎没快多少，更好的算法老师说参见《算法导论》。

    下面是\(O(n)\) 求第k大的思路：

    我们可以借助快速排序的思想解决这个问题。考虑快速排序的划分过程，在快速排序的「划分」结束后，数列 \(A_{p} \cdots A_{r}\) 被分成了 \(A_{p} \cdots A_{q}\) 和 \(A_{q+1} \cdots A_{r}\)，此时可以按照左边元素的个数（\(q - p + 1\)）和 \(k\) 的大小关系来判断是只在左边还是只在右边递归地求解。 可以证明，在期望意义下，程序的时间复杂度为 \(O(n)\)。

    多叉堆

    二叉堆不一定效率最高，老师提到四叉堆效率可能更高，但具体实现还是二叉堆更普遍。
